# PettahPro Build Roadmap

Live tracker of what's shipped, what's next, and what's backlog ŌĆö cross-checked against the 23 spec files in `/docs/` (see [`_summary.md`](./_summary.md) for a per-file digest).

**Three-doc triangle:** this file says what's shipped; [`_status.md`](./_status.md) says what's broken, fragile, or at-risk right now; [`_gaps.md`](./_gaps.md) is the parking lot for real gaps that aren't on the roadmap yet (security, cost-centers, bank feeds, e-filing, observability, platform workstreams). Promote items from `_gaps.md` into this file when it's their turn.

Last updated: 2026-04-23 after PR #82 "#35 kit/bundle items" (follows #81 document attachments, #80 purchase requisitions, #79 final-settlement routing, #78 payroll+bonus, #77 POs, #76 bills, #75 expense claims, #74 engine core + JE). **All must-haves, all original should-haves, #28 POS, #29 commission engine, #45 digest windows, #31 customer portal (+ hardening), #36 inventory categories, the full approval engine (#43 + #43aŌĆō#43e), #30 purchase requisitions, #32 document attachments, and now #35 kit/bundle items all shipped.** Bundles are the second nice-to-have off the M/L pile. Lightweight, no-BOM assembly: a bundle is a virtual SKU with a static component list; selling one explodes into per-component stock issues at invoice post time, rolling weighted-average COGS across components into a single JE.

SECURITY DEFINER helpers (`portal_find_customers_by_email`, `portal_mint_otp`, `portal_find_otp`, `portal_consume_otp`, `portal_resolve_session`) keep the pre-auth cross-tenant lookups off RLS (same pattern as `auth_find_user_*`). v1 scope: read-only self-serve; pay-online (#31 ┬¦3) explicitly deferred (PR #71)
- **Multi-currency on sales & purchases** ŌĆö invoices, credit notes, bills, debit notes, and both payment sides carry optional `currency` + `fx_rate` + `foreign_total_cents` alongside the ledger-in-LKR `amount_cents`.
